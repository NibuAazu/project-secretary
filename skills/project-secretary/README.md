# project-secretary（案件推進秘書スキル）

複数の案件・プロジェクトを並走管理するための Claude スキル。
「今どの案件が誰待ちで、次に何をいつやるべきか」を整理し続ける秘書として振る舞う。

DXコンサルなど、外部クライアント案件を複数抱えて進める働き方向け。

## できること

- **取り込み** — 打ち合わせの文字起こし・メール・資料から、決定事項 / アクション /
  🚩保留・要確認フラグ / 未回答の問い合わせ を抽出して案件台帳を更新
- **現状照会** — 「◯◯の件どうなってる?」にステータス・直近・次の一手・誰待ちで回答
- **フォロー漏れの指摘** — 相手待ちで放置されている件、確認待ちで止まっている件を能動的に検知
- **段取り出し** — 全案件を横断して「今日やること」と「相手待ちの件」を分けて提示

設計の核は、確認が取れていないことを勝手に確定させない点。
迷ったら決定事項ではなく🚩保留フラグに倒し、後工程（仕様書・コーディング指示）に
誤りを流し込まないようにしている。

## 構成

```
project-secretary/
├── SKILL.md                    # 秘書の振る舞いの定義（本体）
└── assets/
    └── ledger-template.md      # 案件ごとに維持する台帳テンプレート
```

## 使い方

### Claude Code の場合
リポジトリ内の `.claude/skills/` にこのフォルダを置くと、そのプロジェクトで利用可能。
チームや複数マシンで共有・バージョン管理する運用に向く。
（マーケットプレイス登録での配布も可能。最新手順は公式ドキュメント参照:
https://docs.claude.com/en/docs/claude-code/overview ）

### Claude.ai（web / アプリ）の場合
`.skill` ファイルにパッケージして設定画面からアップロードする。
このリポジトリで中身を管理し、更新したら再パッケージしてアップロードし直す運用になる。
（機能の入口: https://support.claude.com ）

## 台帳の置き場所

案件台帳（`ledger-template.md` から起こす1枚）は、案件ごとに1ファイルで管理する。
Google Drive、リポジトリ、Claudeのプロジェクト機能など、運用に合わせて選ぶ。
